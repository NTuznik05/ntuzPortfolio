General Reflection

Being completely new to almost any type of coding, I initially found a lot of the HTML website design to be quite overwhelming. A lot of it was extremely confusing, especially when I would be receiving errors because of the slightest syntax issues, which is, to be expected, but still a little frustrating. Once I began to grasp HTML, It actually became quite fine, although my website is quite simple, I enjoyed implementing different paragraphs and blurbs etc. It’s fun to be able to personalize it and such. I also just like the simplistic nature of HTML things, or at least, that’s how mine looks. That being said, I was SOO lost in the style sheet at first as well, but, like HTML, once I got the hang of it, it became quite fun. When it comes to HTML vs WordPress? I think that WordPress is an absolute cakewalk. It took almost no effort to set up in comparison to HTML, but that being said, I think HTML probably has more means of customization if you put a lot of time into it. WordPress is simple because you can just borrow someone else’s theme. But, I think both websites require a lot of patience and creativity, both take time to get settled in with, just in different ways. HTML takes a lot more knowledge of general coding and stuff like that, especially when you take into consideration the communication with Git Bash. I had a lot of fun trying to follow along with GitBash in class, super thankful for the notes I took, otherwise I absolutely would’ve been lost trying to push and commit files. I hope to improve with my understanding of HTML in general, I find myself at roadblocks and constantly having to search or check back on things, which maybe I’d prefer if I understood it like the back of my hand.
